# ERPNext 新一代承兑汇票管理模块 — 需求文档

> **版本**: v1.0  
> **日期**: 2026-04-07  
> **背景**: 基于上海票据交易所「中国票据业务系统」（新一代票据系统），在 ERPNext 中实现电子商业承兑汇票的全生命周期管理。

---

## 1. 项目背景

### 1.1 业务背景

2024年7月，原电子商业汇票系统（ECDS）正式下线，所有票据业务迁移至上海票交所建设的「中国票据业务系统」（新一代票据系统）。新系统最大的变化是引入了**票据包号 + 子票区间**的概念，实现了票据的"找零支付"功能，即一张大额票据可以按需拆分为多个子票，分别用于不同的支付场景。

### 1.2 系统现状

ERPNext 标准模块不包含中国承兑汇票管理功能。当前企业通常在线下或通过 Excel 手工管理票据台账，存在以下痛点：

- 票据状态跟踪困难，容易遗漏到期票据
- 子票拆分后难以追踪各子票流向
- 票据操作无法自动生成会计凭证，需手工录入日记账
- 背书链条不透明，审计追溯困难

### 1.3 建设目标

在 ERPNext 上开发自定义应用（Frappe Custom App），实现承兑汇票的标准化管理，覆盖四大核心业务：**背书接收、背书转让、提前贴现、到期兑付**，并完整支持新一代票据系统的**子票拆分**机制。

## 2. 票据编号规则

### 2.1 票据包号

新一代票据系统的票据包号共 **30位阿拉伯数字**，结构如下：

| 位数      | 含义             | 说明                                                       |
| --------- | ---------------- | ---------------------------------------------------------- |
| 第1位     | 票据种类标识     | 5=银行承兑汇票，6=商业承兑汇票，7=供应链商票，8=供应链银票 |
| 第2-13位  | 支付系统行号     | 出票人开户行的支付系统行号（12位）                         |
| 第14-21位 | 出票信息登记日期 | 票交所成功登记日期（8位，YYYYMMDD）                        |
| 第22-29位 | 当天流水号       | 票交所生成的当天唯一流水号（8位）                          |
| 第30位    | 校验码           | 双模校验码算法生成                                         |

### 2.2 子票区间

- 子票区间由**起始序号**和**结束序号**组成
- 每一个子票序号对应 **0.01元**（即标准金额）
- 票据金额 = 子票张数 × 0.01元
- 示例：子票区间 `1-1000000` 表示金额为 **10,000.00元**
- 子票区间为 `0` 表示该票据**不可拆分**

### 2.3 拆分规则

- 拆分时票据包号不变，子票区间随拆分变化
- 所有拆分后的子票区间之和必须等于原票面额
- 拆分层级（所有持票人总和）上限为 **500人**
- 可拆分的业务环节：背书、贴现、质押申请、质押解除、冻结、追索
- 不可拆分的环节：出票签发阶段（出票/承兑/收票/撤票）、到期处理阶段（提示付款/线上追索）、贴现后银行间交易、再贴现

## 3. 功能需求

### 3.1 票据台账管理（核心）

**描述**: 作为系统的核心 DocType，记录每张票据/子票的完整信息和当前状态。

**主要字段**:

- 票据包号（30位，唯一标识）
- 票据种类（银票/商票/供应链银票/供应链商票）
- 子票区间起始号、子票区间结束号
- 票面金额（自动根据子票区间计算）
- 出票日期、到期日期
- 出票人名称、出票人账号、出票人开户行
- 承兑人名称、承兑人账号、承兑人开户行
- 收款人名称
- 当前持票人
- 票据状态（见状态机设计）
- 流通标志（可流通 / 不可转让 / 已结束）
- 是否可拆分（是/否）
- 不得转让标记（是/否）
- 父票据引用（拆分后指向原票据）
- 关联销售发票 / 采购发票

**票据状态机**:

```
已签发 → 已收票-可流通 → 背书待签收 → 已背书转让
                        → 贴现待确认 → 已贴现
                        → 提示付款中 → 已结清-已结束
```

**需求要点**:

- 支持按票据包号、承兑人、到期日期、状态等多维度查询和筛选
- 提供票据台账总览看板，按状态分类统计
- 到期日临近提醒（可配置提前天数）
- 支持从销售发票、采购发票快速创建或关联票据

### 3.2 背书接收

**描述**: 企业收到客户以承兑汇票支付的货款时，登记票据信息并签收。

**业务流程**:

1. 从销售发票或独立入口创建票据接收记录
2. 录入票据包号、子票区间、票面金额等票据正面信息
3. 录入背书人（即付款客户）信息
4. 确认签收，票据状态变为「已收票-可流通」
5. 系统自动生成会计凭证

**会计分录**:

| 科目     | 借方     | 贷方     |
| -------- | -------- | -------- |
| 应收票据 | 票面金额 | —        |
| 应收账款 | —        | 票面金额 |

**需求要点**:

- 签收时校验票据包号格式（30位数字，首位为5/6/7/8）
- 子票区间为 `0` 时提示「该票据不可拆分」
- 自动关联对应的销售发票并核销应收账款
- 支持批量接收（同一客户多张票据）

### 3.3 背书转让

**描述**: 企业将持有的承兑汇票背书转让给供应商，用于支付采购货款。支持部分金额转让（即子票拆分）。

**业务流程**:

1. 选择要转让的票据（从票据台账中筛选状态为「已收票-可流通」的票据）
2. 选择被背书人（供应商）
3. 输入转让金额
4. **如转让金额 < 票面金额且票据可拆分**：
   - 系统自动计算拆分方案（按0.01元/张计算拆分点）
   - 用户确认拆分方案
   - 原票据标记为「已拆分」
   - 生成两条（或多条）新的子票记录
   - 转让部分状态变为「背书待签收」
   - 剩余部分状态保持「已收票-可流通」
5. **如转让金额 = 票面金额**：整票转让，无需拆分
6. 系统自动生成会计凭证

**子票拆分算法**:

```
输入: 原票子票区间 [start, end], 转让金额 amount
处理:
  1. 校验 amount 是否为 0.01 的整数倍
  2. 计算拆分张数: split_count = amount / 0.01
  3. 生成子票A: [start, start + split_count - 1], 金额 = amount
  4. 生成子票B: [start + split_count, end], 金额 = 原金额 - amount
  5. 校验: 子票A金额 + 子票B金额 = 原票面金额
  6. 校验: 当前拆分层级总人数 ≤ 500
```

**会计分录**:

| 科目     | 借方     | 贷方     |
| -------- | -------- | -------- |
| 应付账款 | 转让金额 | —        |
| 应收票据 | —        | 转让金额 |

**需求要点**:

- 拆分前校验票据是否可拆分（子票区间不为0）
- 拆分后的子票继承原票据包号，仅子票区间不同
- 记录完整的背书链条（背书人 → 被背书人 → 背书日期 → 子票区间）
- 支持「不得转让」标记及撤销功能
- 拆分金额输入时提供可视化子票区间预览

### 3.4 提前贴现

**描述**: 企业将未到期的承兑汇票向银行申请贴现，提前获取资金。支持整票或部分子票贴现。

**业务流程**:

1. 选择要贴现的票据（支持选择全部或部分子票区间）
2. 输入贴现银行、贴现利率（年化）
3. 系统自动计算贴现利息和实际到账金额：
   - 贴现利息 = 票面金额 × 贴现利率 × 剩余天数 ÷ 360
   - 实际到账金额 = 票面金额 - 贴现利息
4. 如仅贴现部分金额，触发子票拆分流程
5. 确认提交后票据状态变为「已贴现」
6. 系统自动生成会计凭证

**会计分录**:

| 科目                  | 借方         | 贷方     |
| --------------------- | ------------ | -------- |
| 银行存款              | 实际到账金额 | —        |
| 财务费用-票据贴现利息 | 贴现利息     | —        |
| 应收票据              | —            | 票面金额 |

**需求要点**:

- 自动计算剩余天数（到期日 - 贴现日，遇节假日顺延规则可配置）
- 贴现利率支持手动输入，也可从银行利率表中选取
- 贴现后票据不可再次背书转让
- 支持按贴现银行统计贴现汇总

### 3.5 到期兑付

**描述**: 票据到期后，承兑人（或承兑人开户行）向持票人支付票面金额。

**业务流程**:

1. 系统定时任务每日扫描即将到期的票据
2. 到期前N天（可配置，默认7天）自动发送提醒通知
3. 到期日系统自动生成「提示付款」记录
4. 银行确认资金到账后，操作员确认兑付
5. 票据状态变为「已结清-已结束」
6. 系统自动生成会计凭证

**会计分录**:

| 科目     | 借方     | 贷方     |
| -------- | -------- | -------- |
| 银行存款 | 票面金额 | —        |
| 应收票据 | —        | 票面金额 |

**需求要点**:

- 银票到期系统自动发起提示付款（模拟新一代系统自动处理）
- 商票到期自动发起提示付款申请，需等待付款人手动应答
- 支持逾期未兑付的追索流程记录
- 到期兑付支持批量确认（同一到期日多张票据）
- 到期日遇法定节假日自动顺延

## 5. 数据模型设计

### 5.1 DocType 清单

| DocType 名称                  | 类型         | 说明                           |
| ----------------------------- | ------------ | ------------------------------ |
| Bill of Exchange（票据台账）  | 主表，可提交 | 核心票据信息，支持状态机       |
| Bill Sub Ticket（子票记录）   | 子表         | 记录每次拆分后的子票区间       |
| Endorsement Log（背书记录）   | 主表，可提交 | 记录每次背书操作的完整信息     |
| Endorsement Chain（背书链条） | 子表         | 票据台账的子表，记录背书历史链 |
| Bill Discount（贴现记录）     | 主表，可提交 | 记录贴现操作的详细信息         |
| Bill Payment（兑付记录）      | 主表，可提交 | 记录到期兑付的确认信息         |
| Bill Receive（票据接收）      | 主表，可提交 | 记录背书接收的操作信息         |
| Bill Transfer（票据转让）     | 主表，可提交 | 记录背书转让的操作信息         |

### 5.2 核心关联关系

```
Sales Invoice ──→ Bill Receive ──→ Bill of Exchange (创建)
                                        │
                    ┌───────────────┬────┴─────┬──────────────┐
                    ↓               ↓          ↓              ↓
             Bill Transfer    Bill Discount  Bill Payment   子票拆分
             (背书转让)        (提前贴现)     (到期兑付)    (Sub Ticket)
                    │               │          │
                    ↓               ↓          ↓
            Purchase Invoice    Bank Entry   Bank Entry
             (核销应付)        (银行到账)    (银行到账)
```

## 7. 技术架构要求

### 7.1 开发方式

- 以 **Frappe 自定义应用**（Custom App）形式开发，独立于 ERPNext 核心
- 应用名称建议：`bill_management`
- 兼容 ERPNext v14 / v15

### 7.2 关键技术点

| 技术项              | 实现方式                                     |
| ------------------- | -------------------------------------------- |
| 票据状态机          | Frappe Workflow                              |
| 子票拆分算法        | Python Controller（`on_submit` hook）        |
| 会计凭证自动生成    | `make_gl_entries()` API                      |
| 到期自动提醒        | Frappe Scheduler Event（每日定时任务）       |
| 票据台账看板        | Frappe Dashboard / Number Card               |
| 子票区间可视化选择  | Client Script（前端滑块组件）                |
| 与销售/采购发票集成 | Custom Button + Server Script                |
| 票据包号格式校验    | Validate hook（正则校验30位数字 + 首位规则） |

### 7.3 权限设计

| 角色     | 权限                                            |
| -------- | ----------------------------------------------- |
| 出纳     | 创建/提交票据接收、票据转让、贴现记录、兑付确认 |
| 会计     | 查看所有票据记录、审核会计凭证、管理会计科目    |
| 财务经理 | 全部权限 + 审批贴现操作 + 查看报表              |
| 审计     | 只读权限，查看所有记录和背书链                  |

## 8. 非功能需求

- **数据完整性**: 票据拆分后所有子票金额之和必须等于原票面金额，系统需在每次拆分时强制校验
- **审计追踪**: 所有票据操作需记录完整的操作日志（操作人、操作时间、操作类型、操作前后状态）
- **并发控制**: 同一张票据不允许同时进行两个操作（如同时转让和贴现）
- **数据备份**: 票据数据属于财务核心数据，需纳入日常备份策略
- **性能**: 票据台账在 10,000 条记录规模下，查询响应时间 < 2秒
